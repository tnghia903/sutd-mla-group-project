Spatial Feature Extraction

How CNNs convert pixels into spatial understanding

Let $\mathbf{I} \in \mathbb{R}^{H \times W \times 3}$ be a document page image. A convolutional backbone $f_\theta$ maps it to a multi-scale feature hierarchy:

$$\mathbf{F}^{(l)} = f_\theta^{(l)}(\mathbf{I}), \quad l \in \{1, 2, \ldots, L\}$$

where $\mathbf{F}^{(l)} \in \mathbb{R}^{H_l \times W_l \times C_l}$ has decreasing spatial resolution and increasing channel depth as $l$ grows.

Bounding-Box Crop Extraction

Given normalised prediction $(\hat{x}_c, \hat{y}_c, \hat{w}, \hat{h})$ and image dimensions $(W, H)$:

$$x_{\min} = \lfloor (\hat{x}_c - \hat{w}/2) \cdot W \rfloor \qquad x_{\max} = \lfloor (\hat{x}_c + \hat{w}/2) \cdot W \rfloor$$ $$y_{\min} = \lfloor (\hat{y}_c - \hat{h}/2) \cdot H \rfloor \qquad y_{\max} = \lfloor (\hat{y}_c + \hat{h}/2) \cdot H \rfloor$$
$$\mathbf{R} = \mathbf{I}\bigl[y_{\min}:y_{\max},\; x_{\min}:x_{\max},\; :\bigr]$$

LSTM Gate Mechanics

How memory flows through the hidden state transition $\mathbf{h}_\tau \to \mathbf{h}_{\tau+1}$

Interactive: LSTM Cell

Click each gate to see its equation, role, and animation

Select a Gate

Click on any gate in the diagram to explore its function.

The LSTM cell uses four gates to control information flow through the cell state and hidden state.

f Forget Gate
$\mathbf{f}_\tau = \sigma\bigl(\mathbf{W}_f [\mathbf{h}_{\tau-1}; \mathbf{x}_\tau] + \mathbf{b}_f\bigr)$

Decides what to discard from the cell state. Outputs values in $[0,1]$: 0 = completely forget, 1 = fully retain.

i Input Gate
$\mathbf{i}_\tau = \sigma\bigl(\mathbf{W}_i [\mathbf{h}_{\tau-1}; \mathbf{x}_\tau] + \mathbf{b}_i\bigr)$

Decides what new information to store in the cell state from the candidate values.

c̃ Cell Candidate
$\tilde{\mathbf{c}}_\tau = \tanh\bigl(\mathbf{W}_c [\mathbf{h}_{\tau-1}; \mathbf{x}_\tau] + \mathbf{b}_c\bigr)$

Creates candidate values that could be added to the cell state. Values in $[-1, 1]$.

o Output Gate
$\mathbf{o}_\tau = \sigma\bigl(\mathbf{W}_o [\mathbf{h}_{\tau-1}; \mathbf{x}_\tau] + \mathbf{b}_o\bigr)$

Decides what to output as the hidden state, based on the filtered cell state.

Cell State Update

The cell state is the memory highway — it carries long-range information with minimal degradation:

$$\mathbf{c}_\tau = \underbrace{\mathbf{f}_\tau \odot \mathbf{c}_{\tau-1}}_{\text{what to keep}} + \underbrace{\mathbf{i}_\tau \odot \tilde{\mathbf{c}}_\tau}_{\text{what to add}}$$

Hidden State Output

The hidden state is the filtered view of the cell state:

$$\overrightarrow{\mathbf{h}}_{\tau} = \mathbf{o}_\tau \odot \tanh(\mathbf{c}_\tau)$$

CTC Loss

Connectionist Temporal Classification — solving the alignment problem

The Alignment Problem

The model produces $T$ time-step outputs, but the target label $\mathbf{l}$ has $S \leq T$ characters. How do we know which output corresponds to which character?

CTC solves this by introducing a blank token $\epsilon$ and marginalising over all valid alignments.

The Many-to-One Mapping $\mathcal{B}$

Define $\Sigma' = \Sigma \cup \{\epsilon\}$. The mapping $\mathcal{B}$ first removes repeated characters, then strips blanks:

CTC Probability & Loss

The probability of a path, assuming independence between time-steps:

$$P(\boldsymbol{\pi} \mid \mathbf{I}) = \prod_{\tau=1}^{T} y_{\pi_\tau}^\tau$$

Marginalise over all valid alignments to get the label probability:

$$P(\mathbf{l} \mid \mathbf{I}) = \sum_{\boldsymbol{\pi} \in \mathcal{B}^{-1}(\mathbf{l})} \prod_{\tau=1}^{T} y_{\pi_\tau}^\tau$$

The CTC loss minimises negative log-probability:

$$\boxed{\mathcal{L}_{\text{CTC}} = -\ln P(\mathbf{l} \mid \mathbf{I}) = -\ln \sum_{\boldsymbol{\pi} \in \mathcal{B}^{-1}(\mathbf{l})} \prod_{\tau=1}^{T} y_{\pi_\tau}^\tau}$$

Interactive: CTC Forward Lattice

Hover over cells to see the forward variable $\alpha_\tau(s)$ computation. The lattice computes $P(\mathbf{l} \mid \mathbf{I})$ in $\mathcal{O}(T \cdot |\mathbf{l}'|)$ time.

$$\alpha_\tau(s) = \sum_{\substack{\boldsymbol{\pi}_{1:\tau} \\ \mathcal{B}(\boldsymbol{\pi}_{1:\tau}) = \mathbf{l}'_{1:s}}} \prod_{t=1}^{\tau} y_{\pi_t}^t$$

The forward variable accumulates path probabilities using dynamic programming. Hover over lattice cells to explore.

Greedy Decoding

At inference, select the most probable symbol at each time-step:

$$\mathbf{l}^* \approx \mathcal{B}\!\left(\arg\max_{\pi_1} y_{\pi_1}^1,\; \arg\max_{\pi_2} y_{\pi_2}^2,\; \ldots,\; \arg\max_{\pi_T} y_{\pi_T}^T\right)$$
